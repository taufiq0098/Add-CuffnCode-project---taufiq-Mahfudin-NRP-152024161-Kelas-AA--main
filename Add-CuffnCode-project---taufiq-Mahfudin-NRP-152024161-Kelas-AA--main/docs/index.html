Overview

What is CuffnCode?

CuffnCode retrofits a standard blood pressure cuff with custom electronics and open-source software, transforming a commodity medical device into a rich experimental platform.

📚

Teaching Tool

Visualize the oscillometric method in real-time. See exactly how systolic and diastolic pressure are derived from the oscillation envelope — ideal for biomedical engineering courses.

🔬

Research Platform

Swap in custom signal processing algorithms, test alternative BP detection methods, and log every sample for offline analysis. Full access to raw ADC data.

🛠️

Embedded Control

Control a DC micro-pump and two solenoid valves with precise timing. Implement PID deflation control, test valve sequencing strategies, and measure latency.

🐍

Python Ecosystem

Built on NumPy, SciPy, and Matplotlib. Extend with scikit-learn for ML classifiers, or biosppy for advanced biosignal processing.

Hardware

Embedded System Design

The hardware core consists of a microcontroller, a DC micro-pump, two solenoid valves, and a MEMS pressure sensor — all interfaced via a custom switching circuit.

Switching Circuit

Each actuator (pump + 2 valves) is driven by an N-channel MOSFET (IRLZ44N) acting as a low-side switch. A flyback diode (1N4007) protects against inductive kickback from the solenoid coils.

Pressure Sensor

The Freescale MPX5050 is a piezoresistive MEMS sensor rated 0–50 kPa (0–375 mmHg). Its analog output is read by the Arduino's 10-bit ADC and converted to mmHg in Python using a linear calibration: 

voltage = (adc / 1023) × 5.0 V
kPa     = (voltage - 0.2) / (4.7 - 0.2) × 50
mmHg    = kPa × 7.5006

Software

Python Stack

The Python application handles signal acquisition, processing, BP computation, visualization, and data logging — all in a single unified dashboard.

src/simulator.py

Generates realistic synthetic oscillometric waveforms for demo without hardware. Uses a Gaussian amplitude model centered at MAP.

  • Configurable SBP, DBP, HR, noise
  • 3-phase: inflate → plateau → deflate
  • Real-time generator (streaming)
src/signal_processing.py

Butterworth filters and envelope extraction to separate oscillations from DC pressure trend.

  • Bandpass: 0.5–8 Hz (cardiac band)
  • Lowpass: 0.3 Hz (pressure trend)
  • Savitzky-Golay envelope smoothing
  • Heart rate from inter-peak intervals
src/bp_algorithm.py

Oscillometric BP algorithm with literature-validated amplitude ratio thresholds.

  • MAP at peak oscillation amplitude
  • SBP ratio: 0.45 × peak amplitude
  • DBP ratio: 0.70 × peak amplitude
  • AHA 2017 classification
src/acquisition.py

Threaded serial reader for real-time hardware data streaming.

  • 115200 baud, 100 Hz sampling
  • ADC → mmHg calibration
  • Send INFLATE/DEFLATE/STOP commands
  • Thread-safe queue architecture
gui/dashboard.py

Modern dark-themed dashboard with live waveform and metric cards.

  • customtkinter + matplotlib embedded
  • Real-time scrolling waveform (100 Hz)
  • SBP / DBP / MAP / HR metric cards
  • Simulation mode with sliders
src/data_logger.py

Persists measurement sessions for offline analysis.

  • Waveform → CSV (time, pressure, phase)
  • Results → JSON (SBP, DBP, MAP, HR)
  • Session-based filenames (timestamp)
  • Load and replay saved recordings

Quick Start

git clone https://github.com/Student-Embedded-Control-and-AI-Fest/CuffnCode.git
cd CuffnCode
pip install -r requirements.txt
# Launch the GUI dashboard
python main.py

# Click "▶ Start Measurement" to begin simulation
pip install pytest
pytest tests/ -v

# Expected output:
# tests/test_signal_processing.py ........ PASSED
# tests/test_bp_algorithm.py ............ PASSED
from src.simulator import SimConfig, get_sample_waveform
from src.bp_algorithm import compute_bp

# Generate synthetic waveform
config = SimConfig(target_sbp=130, target_dbp=85, heart_rate=70)
t, p, phases = get_sample_waveform(config)

# Compute blood pressure
result = compute_bp(t, p, fs=100.0, phase_arr=phases)
print(result)
# → SBP: 128 mmHg | DBP: 83 mmHg | MAP: 98 mmHg | HR: 70 BPM
Algorithm

Oscillometric Method

CuffnCode implements the oscillometric method — the same principle used in most modern automatic blood pressure monitors.

1

Cuff Inflation

Pump inflates the cuff above systolic pressure. No blood flows → no oscillations detected.

→
2

Controlled Deflation

Valve 1 pulses open in steps of 50 ms, releasing pressure at ~2–3 mmHg/sec.

→
3

Signal Filtering

Bandpass (0.5–8 Hz) isolates cardiac oscillations. Lowpass (0.3 Hz) extracts the DC pressure trend.

→
4

Envelope & MAP

Oscillation envelope extracted via Savitzky-Golay smoothing. MAP = cuff pressure at peak envelope.

→
5

SBP & DBP Detection

SBP at 45% of peak amplitude (before MAP). DBP at 70% of peak amplitude (after MAP).

Oscillation Amplitude vs. Cuff Pressure

The amplitude reaches its peak at MAP. Ratio thresholds (0.45 and 0.70 × peak) determine SBP and DBP. These ratios are empirically derived from clinical validation studies.
